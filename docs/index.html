Release Notes

What's been shipped — written for operators, venue managers, and the curious.

May 12, 2026

Booking request detail now shows intake info & recommended space

Feature

The booking request detail page now surfaces the structured intake data the AI collects during a conversation — all in one place so you never have to dig through the email thread.

  • Booking type — the lead's stated space preference in their own words
  • Duration — how long they need the space
  • Space preference — private, semi-private, or flexible
  • Beverage format — open bar, hosted tab, cash bar, etc.
  • Recommended space — the venue space the AI matched to this lead
  • Lead recap — a one-sentence AI summary of the inquiry

The panel only appears once at least one field has been collected — no empty boxes cluttering the screen.

→ PR #318
May 12, 2026

Venue edit form now exposes all space metadata fields

Feature

You can now configure every field the AI uses to match leads to spaces — directly from the venue edit screen. No more manual database edits.

  • Max guests — upper capacity limit per space
  • Private — checkbox to mark a space as exclusively bookable
  • Duration options — which time slots the space supports (e.g. 2 hrs, all night)
  • Space amenities — comma-separated features like private_bar, stage
  • Venue amenities — venue-wide features shared across all spaces

Values are entered as plain comma-separated text and stored as structured arrays — no JSON editing required.

→ PR #317
May 12, 2026

AI recommends a venue space based on guest count & preferences

Feature

After extracting intake fields, Toaster now scores every venue space against the lead's requirements and picks the best fit automatically.

  • Filters spaces by headcount range (min/max guests)
  • Scores by privacy preference, duration match, and feature overlap
  • Tiebreaks on lowest pricing floor
  • Sets fit_status: not_a_fit when no space qualifies — prompting you to review
→ PR #311
May 12, 2026

AI draft switches to quote mode once all fields are collected

Feature

Previously the AI always replied with a generic "we'll check availability" template. Now it detects when it has everything it needs and shifts into a confirmation tone — summarising the event details and moving toward a quote.

→ PR #302
May 12, 2026

Document ingestion retries automatically on API timeout

Fix Reliability

Uploading a venue knowledge document would sometimes fail permanently when the Unstructured API timed out under load. Those jobs now retry up to 5 times with exponential backoff. If all retries are exhausted, the failure is recorded in the event log for review.

→ PR #316
May 11, 2026

Qualification status & missing fields shown on booking detail

Feature

A new qualification panel on the booking request detail page shows the AI's fit assessment and lists exactly which fields are still missing — so you know at a glance whether a lead is ready to quote or still needs follow-up.

→ PR #303
May 10, 2026

AI drafts now read the full conversation, not just the first email

Feature

The AI draft writer now receives the complete back-and-forth — every inbound message and every sent reply — as context when composing a response. The booking detail page also gained a unified chronological thread view to match.

→ PR #295
May 10, 2026

AI draft reply generated automatically when a booking request arrives

Feature

Every new booking request now gets an AI-drafted reply ready for your review — no manual trigger needed. The draft waits in a Pending review state and is only sent after you approve it.

→ PR #283
May 10, 2026

Emails only marked as read after a draft is created

Fix

Previously every synced email was marked as read in your inbox, making it hard to tell which ones had actually been processed. Now an email is only marked seen when Toaster has successfully created a booking request and a draft reply.
